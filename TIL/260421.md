#### 포켓몬 API 실습
``` javascript
const response = await fetch(API_URL);
const data = await response.json();

const detailPromises = data.results.map(pokemon => 
    fetch(pokemon.url).then(res => res.json())
);
const detailedPokemons = await Promise.all(detailPromises);

setPokemons(detailedPokemons);
```
- 해당 코드에서 data.results는 이런 배열임
    ```javascript
    [
        { name: 'bulbasaur', url: 'https://.../pokemon/1/' },
        { name: 'ivysaur',   url: 'https://.../pokemon/2/' },
        ...
    ]   
    ```
- `.map()` 함수를 이용해 각 url에 fetch 요청을 만듦
- `fetch()`는 결과를 바로 반환하는 것이 아니라 **나중에 완료될 작업(Promise)** 을 반환함
- `detailPromises`는 아래와 같은 상태임

    ```javascript
    [
        Promise(bulbasaur 상세정보 가져오는 중...),
        Promise(ivysaur 상세정보 가져오는 중...),
        ...
    ]
    ```
- `Promise.all()`을 통해서 작업이 모두 완료될 때까지 기다림
    - 20개의 요청을 병렬로 동시에 보내고 전부 완료되면 결과 배열을 줘서 시간을 단축할 수 있음
