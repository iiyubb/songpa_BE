### 학습 주제: React hooks, async

---

### 학습 내용

**반복적인 컴포넌트를 랜더링 하는 방법** 
-	배열의 map을 이용해 반복 컴포넌트를 랜더링 할 수 있다
-	Props 객체의 배열을 map을 활용해 li 요소 배열을 반환할 수 있다
-	식별자 key를 이용해 엘리먼트를 반복할 수 있다
-	useState에 담긴 값을 반복하여 엘리먼트를 출력할 수 있다

**useEffect**
-	함수형 컴포넌트에서 렌더링 된 이후에 수행할 내용이 필요한 경우 사용하는 hooks
-	기본적으로 마운트된 시점과 업데이트된 시점 두 가지 모두 동작한다
-	의존성 배열을 빈 배열로 전달하면 마운트 시점에만 동작한다
-	의존성 배열에 변경을 확인할 state 값을 주면 해당 값이 변할 때에만 useEffect가 동작한다
-	컴포넌트가 마운트 해제되기 전에 메모리 누수나 충돌을 방지하기 위해 clean-up 함수를 이용해 리소스를 제거해야 한다

---

### 느낀 내용

JS 문법을 건너뛰어 용어가 낯설고 어렵지만 반복 출력 및 todo list 실습을 하며 감을 익혔다.
useEffect에서 메모리 누수를 직접 확인하여 리소스 제거의 중요성을 깨달았다.

---

### 활용 방안

배열에서 filter와 callback 함수를 이용해 필요한 컴포넌트만 반복적으로 랜더링 할 수 있어 불필요한 화면 출력을 방지할 수 있다.
Cleanup 함수로 리소스를 제거해 메모리 누수를 방지할 수 있다.
