### 학습 주제

Java 연산자 & 메소드/패키지/API

---

### 학습 내용

**1. 연산자**

- **연산자(operator)**: 데이터를 처리하는 기호 / **피연산자(operand)**: 연산 대상
- **종류별 분류**
  - 산술: `+`, `-`, `*`, `/`, `%`
  - 대입: `=`, `+=`, `-=`, `*=`, `/=`, `%=`
  - 증감: `++`, `--` (전위/후위에 따라 동작 시점 다름)
  - 비교: `==`, `!=`, `>`, `<`, `>=`, `<=` → boolean 반환
  - 논리: `&&`, `||`, `!` (단축 평가 적용)
  - 삼항: `(조건) ? A : B`
- **항 수에 따른 분류**: 단항(`++x`), 이항(`a + b`), 삼항(`x ? a : b`)
- **주의**: 문자열 값 비교는 `==`가 아니라 `.equals()` 사용 (참조 주소 비교 vs 값 비교)
- **단축 평가**
  - `&&`: 앞이 false면 뒤 실행 X
  - `||`: 앞이 true면 뒤 실행 X

**2. 메소드**

- **정의**: 특정 작업을 수행하는 명령문의 집합
- **선언 구조**: `접근제어자 반환타입 메소드명(매개변수) { ... return ...; }`
- **접근 제어자 범위 (넓음 → 좁음)**: `public` → `protected` → `default` → `private`
- **호출 흐름**: 메소드 호출 시 stack 영역에 공간 할당 → 종료 시 LIFO 순서로 해제
- **전달인자(argument)** vs **매개변수(parameter)**: 호출부에서 넘기는 값 vs 받는 변수
- **return**: 호출 위치로 복귀, 반환 타입과 일치해야 하며 `void`인 경우 생략 가능

**3. 패키지 & 임포트**

- **패키지**: 관련 클래스/인터페이스를 묶는 단위, 소문자로 작성
- 풀 클래스명 = `패키지명.클래스명`
- **import**: 다른 패키지의 클래스를 풀네임 없이 사용할 수 있도록 선언
- `static import`: static 메소드/필드를 클래스명 없이 호출 가능

**4. API**

- JDK가 제공하는 미리 구현된 클래스/인터페이스 모음
- **Math 클래스**: static 멤버로 구성 → `Math.메소드()` 형태로 바로 사용

### 느낀 내용

- null 체크 로직을 짤 때 왜 `obj != null && obj.getX()` 순서가 중요한지 명확히 정리됨.
- 문자열 비교에서 `==`와 `.equals()` 차이는 자바 백엔드 개발자에게는 기본 중의 기본인데, JPA Entity의 `equals()`/`hashCode()` 오버라이드 이슈와도 직접 연결되는 주제라 다시 봐도 의미 있었음.
- 메소드 호출 시 stack 메모리 동작은 디버깅이나 재귀 호출 시 StackOverflowError를 이해하는 데 직접적으로 도움이 됨.

---

### 활용 방안

- **코드 리뷰 시**: `equals()` 누락, 단축 평가를 활용한 null-safe 조건 작성에 적용
- **Spring 개발**: 접근 제어자(`private` 필드 + `public` getter, `protected` 메소드의 상속 활용)를 의식적으로 설계
- **CS 면접 대비**: stack/heap 메모리 구조, call stack 흐름, 기본형 vs 참조형 비교 차이 등 기초 개념을 다시 한 번 점검
- **패키지 설계**: 도메인별 패키지 구조(`controller`, `service`, `repository`, `domain`)를 명확히 분리하는 습관 강화
